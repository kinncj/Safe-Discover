# Safe Discover — CLAUDE.md

## Product Overview

Safe Discover is a Kirigami-based package management GUI for CachyOS/Arch Linux. It provides a unified interface for Pacman, AUR (via paru), Flatpak, firmware updates (via fwupd), and KDE add-ons (via KNewStuff). It does **not** use PackageKit — all operations go through direct CLI tool invocation with privilege escalation via PolicyKit.

**Version**: 0.2.0 | **License**: GPL-3.0-or-later | **Language**: C++20 + QML (Qt6/KF6)

## Build & Test

```bash
# Configure + build
cmake -B build && cmake --build build -j$(nproc)

# Run (binary is in build/bin/, not ./bin/)
./build/bin/safe-discover

# Tests (Qt Test framework)
cmake -B build -DBUILD_TESTING=ON && cmake --build build -j$(nproc)
ctest --test-dir build --output-on-failure
```

## Architecture

### Core Layer

- **CommandRunner** (singleton): All process execution. Two modes: `Embedded` (background, output streamed) and `Terminal` (Konsole for interactive). Supports privilege escalation via `pkexec` + helper script, pacman lock detection (`/var/lib/pacman/db.lck`), job tracking with unique IDs, timeouts with graceful termination. Has `busy` property and emits `outputLine` for real-time command output.
- **PackageModel** (abstract QAbstractListModel): Base class for backends. Defines `PackageInfo` struct (name, version, description, repository, installed, installedVersion, size) and virtual interface (`search`, `fetchDetails`, `install`, `remove`). Signals: `detailsReady`, `confirmationRequired`, `operationFinished`. Note: `confirmationRequired` does NOT include the index — QML uses `packageList.currentIndex`.
- **ToolChecker** (singleton): Detects `paru`, `flatpak`, `fwupdmgr`, `konsole` via `QStandardPaths::findExecutable()`. Sidebar actions are conditionally enabled based on these.
- **LogManager** (singleton): In-memory session log + optional file logging to `~/.local/state/safe-discover/logs/YYYY-MM-DD.log`.

### Backends

| Backend | Tool | Search | Details | Install | Remove | Notes |
|---------|------|--------|---------|---------|--------|-------|
| PacmanBackend | pacman | `-Ss` | `-Si`/`-Qi` | `-S` (privileged) | `-Rns` (privileged) | Regex parsing of key-value output |
| AurBackend | paru | `-Ss` | `-Si`/`-Qi` | `-S` (terminal) | 3-step: remove → find orphans → clean orphans | Terminal mode for builds |
| FlatpakBackend | flatpak | `search` | `info`/`remote-info` | `install` | `uninstall` | Tab-delimited parsing, multi-remote |
| FirmwareBackend | fwupdmgr | `get-devices --json` | JSON | `update` (privileged) | N/A | Per-device updates |

### UpdateManager

Orchestrates **sequential** system-wide updates in 4 steps: Pacman → AUR → Flatpak → Firmware. Stops on any error. Tracks progress via `currentStep`/`currentStepLabel` properties.

### Privilege Escalation

- PolicyKit policy: `org.cachyos.safediscover.pkexec.helper` with `auth_admin_keep`
- Helper script (`safe-discover-helper.sh`): Strict whitelist — only allows `pacman -S/-Syu/-Rns` and `fwupdmgr refresh/update/get-devices/get-updates`. All other operations are blocked.
- Helper lookup: Uses `SAFE_DISCOVER_LIBEXECDIR` compile definition (from CMake's `KDE_INSTALL_FULL_LIBEXECDIR`), with fallback to `../../safe-discover-helper.sh` relative to binary for dev builds.

### KDE Add-ons (KNewStuff)

`KdeAddonsPage.qml` shows a category grid. Clicking a category opens `NewStuff.Dialog` for that `.knsrc` file. No custom C++ backend — KNS handles store.kde.org API, downloads, and installation. Categories map to `.knsrc` files in `/usr/share/knsrcfiles/`.

**Important**: Do NOT use `NewStuff.Page` embedded inside another `Kirigami.Page` — it fights with the pageStack and causes rendering issues. Use `NewStuff.Dialog` instead.

### Navigation

Sidebar navigation uses `navigateTo()` in Main.qml which does `pageStack.clear()` + `pageStack.push()` (NOT `pageStack.replace()`). This ensures clean page transitions even when overlays or sub-pages are open. Before clearing, it closes any open `detailsPane` on the current page.

Pages that have a `PackageDetailsPane` must expose it via `property alias detailsPane: detailsPane` so `navigateTo()` can close it.

### Inline Actions

`PackageListView.qml` has `installRequested(int index)` and `removeRequested(int index)` signals from inline ToolButtons on each delegate. The button handler sets `listView.currentIndex = index` before emitting (critical — the confirmation dialog uses `packageList.currentIndex` to determine which package to act on).

### Operation Feedback

`OutputLogPanel.qml` listens to both `LogManager.onNewLogEntry` and `CommandRunner.onOutputLine` for real-time command output. Shows a `BusyIndicator`, "Operation in progress…" label, and indeterminate `ProgressBar` when `CommandRunner.busy`. Auto-expands when output arrives.

## CI/CD

- **build.yml**: Runs on push/PR to main. Builds, tests, validates PKGBUILD with namcap.
- **nightly.yml**: Runs on push to main. Builds x86_64 and aarch64 packages, publishes to nightly release.
- **release.yml**: Runs on release publish. Builds packages, uploads as release assets.
- **aur-publish.yml**: Runs on release publish. Updates `aur/PKGBUILD` version via sed, pushes to AUR.

**CI packaging note**: All `makepkg` commands must use `PKGDEST` explicitly (e.g., `PKGDEST='$PWD'`) because the default `PKGDEST` may differ across container images. The `BUILDDIR=/tmp/makepkg-build` is used to avoid building in the source tree.

## Version Bumping Checklist

When releasing a new version, update ALL of these:
1. `CMakeLists.txt` — `project(safe-discover VERSION X.Y.Z ...)`
2. `PKGBUILD` — `pkgver=X.Y.Z`
3. `src/main.cpp` — `QStringLiteral("X.Y.Z")` in KAboutData
4. `ca.kinncj.SafeDiscover.metainfo.xml` — add new `<release>` entry
5. `aur/PKGBUILD` and `aur/.SRCINFO` are updated automatically by CI on release

## Conventions

- **i18n domain**: `safe-discover` — use `i18n()` for singular, `i18np()` for plural
- **Sidebar labels**: Use tool names directly (Pacman, AUR, Flatpak, Firmware, KDE Add-ons)
- **Page titles**: Descriptive (e.g., "Pacman Packages", "AUR Packages")
- **Backends**: Inherit `PackageModel`, use `CommandRunner` for all process execution
- **Privilege ops**: Always go through `safe-discover-helper.sh` via `runPrivileged()`
- **QML pages**: Loaded on-demand via `Component` + `navigateTo()` (clear + push)
- **No PackageKit**: Direct CLI tool invocation only
- **Tests**: Tests recompile source files directly (not linked). Any new compile definitions need `#ifndef` fallbacks in the source.
